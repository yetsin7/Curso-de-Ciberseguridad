# Módulo 16: Proyecto Final de Ciberseguridad

Es hora de poner en práctica todo lo aprendido. Realizarás una auditoría de seguridad completa a un entorno empresarial simulado.

## Descripción del Proyecto

El estudiante recibirá acceso a una red empresarial controlada que contiene:
- Un servidor web con múltiples vulnerabilidades.
- Una base de datos con información sensible.
- Un controlador de dominio (Active Directory) con configuraciones débiles.
- Una máquina de usuario final.

## Requisitos del Proyecto

1.  **Fase de Reconocimiento**: Documentar toda la información recolectada de forma pasiva y activa.
2.  **Análisis de Vulnerabilidades**: Identificar y clasificar las debilidades encontradas.
3.  **Explotación**: Ganar acceso inicial a al menos una de las máquinas.
4.  **Escalada de Privilegios y Movimiento Lateral**: Comprometer el controlador de dominio.
5.  **Exfiltración de Prueba**: Demostrar la capacidad de extraer información sensible (sin dañarla).
6.  **Reporte Técnico y Ejecutivo**: Entregar un informe profesional que incluya hallazgos, impacto y recomendaciones de remediación.

## Criterios de Evaluación
- Metodología seguida (OSSTMM / PTES).
- Claridad y profesionalismo en el reporte.
- Calidad de las recomendaciones de seguridad.
