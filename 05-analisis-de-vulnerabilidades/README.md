# Módulo 05 — Análisis de Vulnerabilidades (Explicado Fácil 🔍)

Antes de que un hacker malo entre en tu sistema, tú mismo debes buscar los agujeros. Es como ser un inspector de seguridad que revisa un edificio para ver si las alarmas funcionan y si las puertas cierran bien.

---

## 🧐 ¿Qué aprenderás aquí?

Aprenderemos a usar herramientas que "escanean" las computadoras en busca de fallos conocidos. Es como tener una lista de todas las formas en que alguien ha robado una casa antes e ir revisando si tu casa tiene esos mismos puntos débiles.

| Concepto | Analogía de la vida real |
| :--- | :--- |
| **Escaneo de Vulnerabilidades** | Un inspector de sanidad revisando la cocina de un restaurante. |
| **Falso Positivo** | El inspector dice que hay fuego, pero solo es el vapor de la sopa. |
| **Falso Negativo** | El inspector dice que todo está bien, pero hay una fuga de gas que no vio. |
| **CVE (Common Vulnerabilities and Exposures)** | El libro gordo de todos los fallos de seguridad que se han descubierto en el mundo. |
| **Pentesting vs Análisis** | El análisis busca los huecos; el pentesting intenta entrar por ellos. |

## 🛂 El Revisor de Aeropuerto
Imagina que el análisis de vulnerabilidades es el escáner de seguridad del aeropuerto:
- Tú pasas tus maletas (tus servidores).
- La máquina busca formas prohibidas (vulnerabilidades).
- Si la máquina pita, el guardia revisa si es una amenaza real (un cuchillo) o algo inofensivo (un cortaúñas).

¡Nuestro objetivo es que nada peligroso pase desapercibido!
